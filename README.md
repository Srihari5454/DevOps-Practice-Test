‚öôÔ∏è Main Components Explained
1. bin/backup.sh

This is the core script that performs all the main tasks.
It:

Reads configuration values from backup.config.

Checks if the source directory exists.

Creates folders for daily, weekly, and monthly backups.

Compresses the source directory into a .tar.gz archive with a timestamp.

Saves it under the correct backup folder (daily/weekly/monthly).

Logs each activity (success, failure, or missing files).

This is the heart of the project ‚Äî it does the automation work.

2. config/backup.config

A configuration file that stores user-defined settings like:

Which folder to back up

Where to store backups

Log file paths

Backup schedule or frequency

This makes the script flexible ‚Äî instead of hardcoding values, users can edit the config file.

3. backups/

All generated backup archives are stored here.
It may contain subfolders like:

daily/ ‚Üí backups made every day

weekly/ ‚Üí backups created every week

monthly/ ‚Üí backups taken at the start of each month

Each backup is timestamped to make identification easier.

4. logs/

This folder stores the log file generated by the backup process.
Example:
backup.log records details such as:

Date and time of the backup

Which directory was backed up

Where it was stored

Whether it succeeded or failed

Logs are useful for auditing and debugging backup issues.

5. restore/

This directory is used for restoring files from backup archives.
When the user runs the script with a restore option, the selected backup file gets extracted here.

6. .gitignore

Ensures that large or unnecessary files (like backups and logs) are not uploaded to GitHub.
This keeps the repository lightweight and clean.

7. README.md

Contains a clear explanation of the project ‚Äî including its purpose, folder structure, setup instructions, and usage examples.
This helps others understand and use your project easily.

üîÑ How the Automation Works

Initialization ‚Äì The script loads all paths and settings from the backup.config file.

Validation ‚Äì It checks whether the source folder exists and if necessary directories are available.

Backup Creation ‚Äì The script compresses the source directory into a .tar.gz file and places it under the appropriate folder (daily/weekly/monthly).

Logging ‚Äì Every step (success, failure, or skipped file) is recorded in backup.log.

Restoration (Optional) ‚Äì If the user specifies a restore command, the backup is extracted back into the restore directory.
